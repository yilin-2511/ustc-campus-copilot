## 1. 项目概述与设计目标

### 1.1 项目定位
- **产品名称**：校园 Copilot（USTC Campus Copilot）
- **赛道**：智能体赛道
- **方向**：校园生活场景
- **目标用户**：大一新生（主要），大二及以上学生（次要）
- **核心价值**：通过多智能体协作 + RAG 知识库，为新生提供一站式校园生活智能助手

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **智能交互** | 自然语言对话，理解用户意图，提供精准回答 |
| **多智能体协作** | 专业化分工，各司其职，协同完成任务 |
| **多源信息整合** | 聚合官方平台 + 非正式渠道信息，统一呈现 |
| **个性化推荐** | 基于用户画像，提供定制化建议 |
| **情境感知** | 根据时间、地点、天气等上下文主动服务 |
| **社区驱动** | 用户贡献内容，形成共享知识库 |
| **快速开发** | 使用成熟框架，缩短开发周期 |

### 1.3 技术约束
- **模型资源**：比赛提供 DeepSeek-V4、GLM5.2，每日 50 亿 Token 额度
- **部署平台**：学校本科生算力平台（https://107.ustc.edu.cn/）
- **开发周期**：约 8 周（7月中上旬 - 9月初）
- **团队规模**：2 人

## 3. 技术选型详解

### 3.1 Agent 框架

#### 3.1.1 LangChain ⭐ 核心框架

**简介**：
LangChain 是目前最流行的 LLM 应用开发框架，由 Harrison Chase 于 2022 年创建，GitHub Stars 超过 90k。它提供了一套完整的工具和抽象层，用于构建基于大语言模型的应用程序。

**核心特性**：
- **LLM Chain**：将 LLM 与提示词、输出解析器组合成可复用的链式调用
- **Prompt Templates**：可参数化的提示词模板管理
- **Tool Calling**：内置工具调用机制，支持函数调用（Function Calling）
- **RAG 支持**：完整的检索增强生成 pipeline（文档加载 → 分割 → 嵌入 → 检索 → 生成）
- **Memory**：多种记忆机制（对话历史、实体记忆、向量记忆等）
- **Callbacks**：事件回调系统，支持日志、监控、追踪
- **生态丰富**：与 100+ 个第三方服务集成（向量数据库、文档加载器、嵌入模型等）

**在我们的项目中的用途**：
- 构建每个智能体的核心逻辑
- 管理提示词模板和工具调用
- 实现 RAG 知识库的检索和生成
- 管理对话历史和用户上下文

**关键组件**：
```python
# LangChain 核心组件示例
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.tools import tool
from langchain.agents import create_tool_calling_agent, AgentExecutor

# 定义工具
@tool
def search_knowledge_base(query: str) -> str:
    """搜索校园知识库"""
    ...

# 创建 Agent
agent = create_tool_calling_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools)
```

**优势**：
- 社区活跃，文档完善，遇到问题容易找到解决方案
- 与比赛提供的模型（DeepSeek-V4、GLM5.2）兼容良好
- 内置 RAG 支持，减少重复造轮子
- 工具调用机制成熟，适合多工具场景

**注意事项**：
- API 在 v0.x 到 v1.x 之间有较大变化，需关注版本兼容性
- 链式调用可能导致 Token 消耗较大，需注意配额管理

#### 3.1.2 CrewAI ⭐ 多智能体协作

**简介**：
CrewAI 是一个专注于多智能体协作的框架，由 João Moura 创建，GitHub Stars 超过 25k。它模拟真实团队的工作方式，让多个智能体各司其职、协同完成任务。

**核心特性**：
- **Agent（角色）**：定义智能体的角色、目标、 backstory（背景故事）
- **Task（任务）**：定义任务描述、预期输出、执行 Agent
- **Crew（团队）**：将多个 Agent 和 Task 组织成协作团队
- **Process（流程）**：支持顺序执行（Sequential）和自主执行（Hierarchical）
- **Memory**：团队级记忆，支持短期和长期记忆
- **Delegation**：Agent 之间可以委托任务
- **Planning**：内置规划能力，自动分解复杂任务

**在我们的项目中的用途**：
- 组织学习助手、生活助手、社交助手等智能体
- 定义每个智能体的角色和职责
- 管理任务分配和协作流程
- 实现智能体间的任务委托和信息共享

**关键组件**：
```python
from crewai import Agent, Task, Crew, Process

# 定义 Agent
learning_agent = Agent(
    role="学习助手",
    goal="帮助学生规划学习，提供课程和图书馆资源信息",
    backstory="你是中科大的学习助手，熟悉所有课程和图书馆资源...",
    tools=[course_tool, library_tool, rag_tool],
    llm=llm
)

life_agent = Agent(
    role="生活助手",
    goal="帮助学生解决日常生活问题，推荐食堂和提供生活指引",
    backstory="你是中科大的生活助手，了解所有食堂和校园服务...",
    tools=[canteen_tool, navigation_tool, repair_tool],
    llm=llm
)

# 定义 Task
task1 = Task(
    description="查询用户下周课表，分析空闲时间",
    expected_output="下周课表总结和空闲时间段列表",
    agent=learning_agent
)

task2 = Task(
    description="根据用户口味偏好推荐食堂",
    expected_output="食堂推荐列表，包含菜品和价格",
    agent=life_agent
)

# 创建 Crew
crew = Crew(
    agents=[learning_agent, life_agent],
    tasks=[task1, task2],
    process=Process.sequential  # 或 Process.hierarchical
)

result = crew.kickoff(inputs={"user_query": "帮我规划下周"})
```

**优势**：
- 专为多智能体协作设计，API 简洁直观
- 角色定义清晰，适合我们"专业化分工"的架构
- 支持任务委托和团队记忆
- 学习曲线较低，快速上手

**注意事项**：
- 相对 LangChain 较新，社区资源较少
- 复杂场景下可能需要自定义流程

#### 3.1.3 LangGraph（备选/补充）

**简介**：
LangGraph 是 LangChain 团队开发的图状态机框架，用于构建复杂的有状态、多参与者的应用程序。它以图为模型，节点（Node）表示操作，边（Edge）表示控制流。

**核心特性**：
- **图状态机**：用图结构定义工作流，支持循环、分支、并行
- **状态管理**：每个节点可以读写共享状态
- **检查点（Checkpointing）**：支持状态持久化和恢复
- **人类在环（Human-in-the-loop）**：支持人工审核和干预
- **流式输出**：支持实时流式响应

**在我们的项目中的用途**：
- 实现复杂的多步骤工作流（如信息聚合 → 去重 → 排序 → 推送）
- 管理智能体间的状态传递
- 实现需要人工审核的流程（如内容审核）

**关键组件**：
```python
from langgraph.graph import StateGraph, END

# 定义状态
class AgentState(TypedDict):
    user_query: str
    intent: str
    tasks: list
    results: list
    final_answer: str

# 构建图
workflow = StateGraph(AgentState)

# 添加节点
workflow.add_node("classify_intent", classify_intent_node)
workflow.add_node("decompose_tasks", decompose_tasks_node)
workflow.add_node("execute_learning", execute_learning_node)
workflow.add_node("execute_life", execute_life_node)
workflow.add_node("synthesize", synthesize_node)

# 添加边
workflow.add_edge("classify_intent", "decompose_tasks")
workflow.add_conditional_edges("decompose_tasks", route_tasks)
workflow.add_edge("execute_learning", "synthesize")
workflow.add_edge("execute_life", "synthesize")
workflow.add_edge("synthesize", END)

app = workflow.compile()
```

**优势**：
- 精确控制执行流程，适合复杂工作流
- 状态管理完善，支持持久化
- 与 LangChain 无缝集成

**注意事项**：
- 学习曲线较陡，需要理解图状态机概念
- 对于简单场景可能过于复杂

#### 3.1.4 AutoGen（备选）

**简介**：
AutoGen 是微软研究院开发的框架，专注于构建可对话的多智能体系统。它通过定义智能体之间的对话模式来实现复杂任务。

**核心特性**：
- **可对话智能体**：智能体通过对话协作
- **代码执行**：内置代码执行环境，智能体可以编写和执行代码
- **人类参与**：支持人类作为智能体参与对话
- **群聊模式**：支持多智能体群聊协作

**在我们的项目中的用途**：
- 如果需要智能体间深度对话协作的场景
- 需要代码执行能力的场景（如数据分析）

**优势**：
- 微软背书，学术背景强
- 代码执行能力独特

**注意事项**：
- 对话模式可能不适合我们的"任务分解 → 并行执行 → 结果整合"架构
- 相对较重，资源消耗较大

**结论**：
- **主选**：LangChain + CrewAI（LangChain 提供基础能力，CrewAI 提供多智能体编排）
- **补充**：LangGraph（用于复杂工作流管理）
- **备选**：AutoGen（如需要代码执行能力时考虑）

### 3.2 RAG 引擎

#### 3.2.1 LlamaIndex ⭐ RAG 核心

**简介**：
LlamaIndex（原 GPT Index）是专为 RAG（检索增强生成）设计的框架，由 Jerry Liu 创建，GitHub Stars 超过 35k。它提供了完整的知识库构建、索引、检索 pipeline。

**核心特性**：
- **数据连接器（Data Connectors）**：支持 100+ 数据源（PDF、Web、数据库、API 等）
- **文档加载器（Document Loaders）**：自动解析各种格式文档
- **文本分割器（Text Splitters）**：多种分割策略（按字符、按语义、按段落等）
- **索引结构（Index Structures）**：支持向量索引、关键词索引、知识图谱索引等
- **检索器（Retrievers）**：多种检索策略（向量检索、混合检索、路由检索等）
- **查询引擎（Query Engines）**：支持简单查询、子查询、多步推理查询
- **Postprocessing**：检索结果重排序、去重、过滤

**在我们的项目中的用途**：
- 构建校园知识库（官方文档、学长学姐经验、活动信息等）
- 实现智能问答的检索和生成
- 管理知识库的更新和版本

**关键组件**：
```python
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from llama_index.core.node_parser import SentenceSplitter
from llama_index.embeddings.openai import OpenAIEmbedding
from llama_index.vector_stores.chroma import ChromaVectorStore

# 加载文档
documents = SimpleDirectoryReader("data/knowledge_base").load_data()

# 分割文档
node_parser = SentenceSplitter(chunk_size=1024, chunk_overlap=200)
nodes = node_parser.get_nodes_from_documents(documents)

# 创建索引
embed_model = OpenAIEmbedding(model="text-embedding-3-small")
index = VectorStoreIndex(nodes, embed_model=embed_model)

# 查询
query_engine = index.as_query_engine(similarity_top_k=5)
response = query_engine.query("东区图书馆几点关门？")
print(response.response)
```

**优势**：
- 专为 RAG 设计，功能全面
- 支持多种索引结构和检索策略
- 与 ChromaDB 等向量数据库集成良好
- 文档加载器丰富，支持多种格式

**注意事项**：
- 与 LangChain 有重叠功能，需注意避免重复
- 嵌入模型需要额外配置

#### 3.2.2 LangChain RAG（补充）

**简介**：
LangChain 内置了完整的 RAG 支持，包括文档加载、分割、嵌入、检索和生成。对于简单场景，可以直接使用 LangChain 的 RAG 功能，无需引入 LlamaIndex。

**核心组件**：
```python
from langchain_community.document_loaders import WebBaseLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain.chains import RetrievalQA

# 加载文档
loader = WebBaseLoader("https://www.lib.ustc.edu.cn/about/hours")
documents = loader.load()

# 分割文档
text_splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
splits = text_splitter.split_documents(documents)

# 创建向量存储
vectorstore = Chroma.from_documents(documents=splits, embedding=OpenAIEmbeddings())

# 创建检索器
retriever = vectorstore.as_retriever(search_type="similarity", search_kwargs={"k": 5})

# 创建 RAG 链
qa_chain = RetrievalQA.from_chain_type(llm=llm, retriever=retriever)
response = qa_chain.invoke({"query": "图书馆几点关门？"})
```

**结论**：
- **主选**：LlamaIndex（功能更专业，适合复杂知识库场景）
- **补充**：LangChain RAG（简单场景可直接使用，减少依赖）

### 3.7 地图与位置服务 🆕

> **2026-06-25 新增**：地图是贯穿多个场景的基础能力，不是独立功能。

#### 3.7.1 设计理念

地图服务采用**"Agent 输出结构化位置数据 + 前端渲染"**的分离架构：

```
Agent 回复（含位置 JSON）
       │
       ▼
┌──────────────────────────────────────┐
│          前端地图组件                  │
│  ┌────────────────────────────────┐  │
│  │  Leaflet.js / 高德 JS API      │  │
│  │  - 标记点 (Marker)             │  │
│  │  - 路线 (Polyline)             │  │
│  │  - 区域 (Polygon)              │  │
│  │  - 用户位置 (Geolocation)      │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

Agent 不直接生成地图图片，而是输出结构化数据，由前端渲染。这样：
- Agent 只需知道坐标和地名，不需要处理地图渲染
- 前端可以用任何地图库，灵活替换
- 用户位置隐私在浏览器端保护

#### 3.7.2 地图前端选型

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| **Leaflet.js** | 开源免费、轻量(42KB)、离线可用 | 需要自备瓦片图 | ⭐ MVP |
| 高德地图 JS API | 国内数据全、导航强 | 需要 API Key、在线依赖 | 备选 |
| Mapbox GL JS | 3D 效果好 | 收费、在线依赖 | 不推荐 |

**推荐 Leaflet.js**：
- 开源无限制，比赛演示不依赖外部服务
- 可以用 OpenStreetMap 瓦片或自建校园瓦片
- 配合 GeoJSON 渲染校园建筑轮廓

### 3.13 内容审核

#### 3.12.1 敏感词过滤

**简介**：
使用敏感词库进行内容过滤，防止不当内容发布。

**实现方案**：
```python
import jieba

class ContentFilter:
    def __init__(self, sensitive_words: set):
        self.sensitive_words = sensitive_words
    
    def filter(self, text: str) -> tuple[bool, list]:
        """检查文本是否包含敏感词"""
        words = jieba.lcut(text)
        found = [w for w in words if w in self.sensitive_words]
        return len(found) == 0, found

# 使用
filter = ContentFilter(load_sensitive_words())
is_clean, found_words = filter.filter(user_content)
```

**实现**：
```python
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from llama_index.core.node_parser import SentenceSplitter
from llama_index.embeddings.openai import OpenAIEmbedding

# 1. 加载文档
documents = SimpleDirectoryReader("knowledge_base/official").load_data()

# 2. 分割文档
node_parser = SentenceSplitter(chunk_size=1024, chunk_overlap=200)
nodes = node_parser.get_nodes_from_documents(documents)

# 3. 向量化
embed_model = OpenAIEmbedding(model="text-embedding-3-small")

# 4. 构建索引
index = VectorStoreIndex(nodes, embed_model=embed_model)

# 5. 存储
index.storage_context.persist(persist_dir="./chroma_db")
```

from abc import ABC, abstractmethod
from typing import List
from datetime import timedelta

class BaseNewsSource(ABC):
    """所有信息源的抽象基类"""
    name: str = "base"
    enabled: bool = True

    def fetch_recent(self, hours: int = 24) -> List[NewsItem]:
        """获取最近 N 小时的新内容"""
        if not self.enabled:
            return []
        try:
            return self._fetch(hours)
        except Exception as e:
            print(f"[news/{self.name}] 采集失败：{e}")
            return []

    @abstractmethod
    def _fetch(self, hours: int) -> List[NewsItem]:
        """子类实现具体采集逻辑"""
